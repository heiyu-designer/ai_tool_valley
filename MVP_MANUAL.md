# AI 工具集合站 - MVP 手册

## 1. 产品概述

### 项目定位
一个简单、专注的 AI 工具导航站，帮助用户发现和管理实用的 AI 工具。

### 目标用户
- 个人用户：需要快速找到合适的 AI 工具
- 站点所有者：通过工具集合站沉淀用户邮箱，便于后续运营引流

### 核心价值
- 聚合优质 AI 工具，按功能分类展示
- 通过邮箱注册方式积累用户资源

## 2. 功能清单

### 2.1 用户系统

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 邮箱注册 | 输入邮箱，获取验证码，设置密码 | P0 |
| 登录 | 邮箱 + 密码登录 | P0 |
| 会话保持 | 关闭浏览器后一段时间内保持登录状态 | P0 |

### 2.2 工具管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 查看工具列表 | 卡片式展示所有工具 | P0 |
| 分类筛选 | 按分类查看工具 | P0 |
| 搜索工具 | 通过名称/描述搜索 | P0 |
| 添加工具 | 添加新的 AI 工具 | P0 |
| 编辑工具 | 修改已有工具信息 | P0 |
| 删除工具 | 删除不需要的工具 | P0 |

### 2.3 工具详情

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 点击跳转 | 登录用户点击工具卡片跳转外链 | P0 |
| 封面自动获取 | 自动抓取工具网站的 favicon 作为封面 | P1（降级方案：用默认图标） |
| 复制链接 | 一键复制工具链接 | P2 |

## 3. 数据结构

### 3.1 用户表 (users)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| email | TEXT | 邮箱，唯一 |
| password | TEXT | 加密后的密码 |
| created_at | DATETIME | 注册时间 |

### 3.2 工具表 (tools)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| name | TEXT | 工具名称 |
| url | TEXT | 工具链接 |
| icon | TEXT | 封面图标 URL |
| category | TEXT | 分类 |
| description | TEXT | 工具描述 |
| pricing | TEXT | 免费/付费标签 |
| created_at | DATETIME | 添加时间 |

### 3.3 分类表 (categories)

预定义四个分类：
- `productivity` - 办公提效
- `coding` - 代码编程
- `content` - 自媒体
- `chat` - AI聊天

### 3.4 验证码表 (verification_codes)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| email | TEXT | 邮箱 |
| code | TEXT | 6位验证码 |
| expires_at | DATETIME | 过期时间 |
| used | BOOLEAN | 是否已使用 |

## 4. 页面结构

```
/                    # 首页（需要登录）
  - 工具卡片网格展示
  - 顶部：Logo + 搜索框 + 用户信息
  - 侧边/顶部：分类筛选
  - 底部：分页（如果工具多的话）

/login               # 登录页（公开）
  - 邮箱 + 密码表单
  - 注册跳转链接

/register            # 注册页（公开）
  - 步骤1：输入邮箱 → 发送验证码
  - 步骤2：输入验证码 + 设置密码 → 注册成功

/manage              # 管理页（需要登录）
  - 工具列表（带编辑/删除按钮）
  - 添加工具表单

/manage/add         # 添加工具页
/manage/edit/[id]   # 编辑工具页
```

## 5. 技术方案

### 5.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | Next.js 14+ (App Router) | React 框架 |
| 语言 | TypeScript | 类型安全 |
| 样式 | Tailwind CSS | 快速样式开发 |
| 数据库 | SQLite + better-sqlite3 | 轻量，无需单独服务 |
| 认证 | NextAuth.js v5 | 用户认证 |
| 邮件 | Resend | 发送验证码邮件 |
| 图标 | Lucide React | 图标库 |

### 5.2 项目结构

```
ai_tool_vally/
├── src/
│   ├── app/                 # Next.js App Router
│   │   ├── (auth)/         # 认证相关页面
│   │   │   ├── login/
│   │   │   └── register/
│   │   ├── (main)/         # 需要登录的页面
│   │   │   ├── page.tsx     # 首页
│   │   │   └── manage/      # 管理页面
│   │   └── api/            # API 路由
│   │       ├── auth/
│   │       ├── tools/
│   │       └── verification/
│   ├── components/         # 组件
│   │   ├── ui/             # 基础 UI 组件
│   │   ├── ToolCard.tsx    # 工具卡片
│   │   ├── ToolForm.tsx    # 工具表单
│   │   └── CategoryFilter.tsx
│   ├── lib/                # 工具函数
│   │   ├── db.ts           # 数据库连接
│   │   ├── auth.ts         # 认证配置
│   │   └── email.ts        # 邮件发送
│   └── types/              # 类型定义
├── data/                   # SQLite 数据库文件
│   └── tools.db
├── prisma/                 # Prisma ORM（管理数据库）
│   └── schema.prisma
└── package.json
```

### 5.3 部署方案

| 方案 | 说明 |
|------|------|
| Docker | 容器化部署，一个命令启动 |
| Vercel | 官方支持，但 SQLite 需要额外配置（不推荐） |
| 手动部署 | 买服务器，SSH 部署 |

**推荐：Docker 部署**，简单可靠。

### 5.4 Docker 配置

```dockerfile
FROM node:20-alpine

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .

# 数据库文件通过 Volume 映射
VOLUME ["/app/data"]

EXPOSE 3000
CMD ["npm", "run", "dev"]
```

启动命令：
```bash
docker run -v ./data:/app/data -p 3000:3000 ai_tool_vally
```

## 6. 设计规范

### 6.1 配色方案

| 用途 | 颜色 | 说明 |
|------|------|------|
| 主色 | `#2563EB` | 蓝色，科技感 |
| 背景 | `#F8FAFC` | 浅灰白 |
| 卡片背景 | `#FFFFFF` | 白色 |
| 文字主色 | `#1E293B` | 深灰 |
| 文字次色 | `#64748B` | 中灰 |
| 边框 | `#E2E8F0` | 浅灰边框 |

### 6.2 布局

- **首页**：响应式网格，卡片 1-4 列自适应
- **管理页**：列表视图，左侧操作按钮
- **间距**：基于 4px 网格（8px, 16px, 24px, 32px）

### 6.3 字体

- 主字体：`Inter`（Google Fonts）
- 代码：`JetBrains Mono`（如有需要）
